QorellyMobile is the cross-platform companion to the Qorelly web dashboard. Built with Flutter, it runs natively on **iOS**, **Android**, **tablets and iPads**, and **desktop** (macOS, Windows, Linux) — all from a single codebase.

## Supported platforms

| Platform | Devices | Status |
|----------|---------|--------|
| iOS | iPhone, iPad, iPad Pro | Available |
| Android | Phones, tablets | Available |
| macOS | MacBook, iMac, Mac Mini | Available |
| Windows | Desktop PCs, laptops | Available |
| Linux | Desktop PCs, laptops | Available |

See [Supported Platforms](/mobile/supported-platforms) for detailed information on each platform and device-specific features.

## Key highlights

- **Cross-platform** — One app for phones, tablets, iPads, and desktop
- **Offline-first** — Create orders without internet and sync automatically when back online
- **Receipt printing** — Print via Bluetooth thermal printers or native POS hardware
- **Barcode scanning** — Scan product barcodes with the device camera for quick POS entry
- **6 operational modes** — Customize the interface for your specific workflow
- **Multi-location** — Switch between branches seamlessly
- **Adaptive layouts** — Optimized UI for every screen size, from phone to desktop

## Getting started

<Steps>
  <Step title="Install the app">
    - **iPhone / iPad** — Download from the App Store
    - **Android phones and tablets** — Download from the Google Play Store
    - **Desktop** — Download the macOS, Windows, or Linux build from [qorelly.com](https://qorelly.com)
  </Step>
  <Step title="Log in">
    Use the same email and password as your Qorelly web account.
  </Step>
  <Step title="Select your mode">
    On first login, the app recommends an [operational mode](/mobile/app-modes) based on your business type. You can accept the recommendation or choose a different mode.
  </Step>
  <Step title="Start working">
    The app opens to your mode's default screen — typically the POS for order-based businesses or the Bookings calendar for appointment-based businesses.
  </Step>
</Steps>

## Shared backend

QorellyMobile shares the same backend as the Qorelly web platform. This means:

- All data syncs between web and the app in real time
- Team members, roles, and permissions are consistent across all platforms
- Orders created on any device appear everywhere else
- Customer profiles are shared across all platforms

## Navigation

The app uses a bottom tab bar on phones and tablets, with the visible tabs depending on your selected [app mode](/mobile/app-modes).

On desktop, the navigation adapts to a sidebar layout that makes full use of the larger screen.

The **More** menu provides access to additional features:

- Menu Management
- Team
- Reports
- Settings
- Work Periods
- Locations
- Help & Support

## Adaptive layouts by screen size

The app automatically adjusts its layout based on screen size:

| Screen size | Device examples | Layout |
|-------------|----------------|--------|
| Small (< 600px) | iPhones, Android phones | Single-column, bottom tab bar |
| Medium (600px–1100px) | iPads, Android tablets | Multi-column grids, split views, enhanced calendar |
| Large (> 1100px) | Desktop, iPad Pro landscape | Full multi-panel layout, sidebar navigation |

### Phone layout
- Single-column interface optimized for one-handed use
- Bottom tab bar for primary navigation
- Swipe gestures for common actions

### Tablet and iPad layout
- Multi-column grids for menu items (4–5 columns)
- Split-view screens for order management
- Larger floor plan view for table management
- Enhanced bookings calendar with more time slots visible
- Side-by-side order creation and item browsing

### Desktop layout
- Full sidebar navigation
- Multi-panel views with maximum information density
- Keyboard shortcuts for power users
- Larger data tables and charts

## App flavors

QorellyMobile is available in three flavors for development and testing:

| Flavor | Purpose | Indicator |
|--------|---------|-----------|
| Production | Live app for real use | No banner |
| Development | For testing new features | Red "DEV" banner |
| Sandbox | Demo mode with sample data | Amber "DEMO" banner |

Each flavor connects to its own backend instance, so test data never mixes with production data.
